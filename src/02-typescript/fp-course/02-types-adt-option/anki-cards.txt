Что такое тип с точки зрения теории множеств?	Тип — это множество допустимых значений. Например, <code>boolean</code> = {true, false} (мощность 2), <code>never</code> = пустое множество (мощность 0), <code>unknown</code> = универсальное множество.	fp::types::adt
Что такое Product type? Как вычислить его мощность?	Product type комбинирует типы через "И" — значение содержит ВСЕ поля. Мощность = произведение мощностей всех полей. Пример: <code>{ mode: 'a' | 'b'; flag: boolean }</code> → 2 × 2 = 4 значения. Формы: interface, tuple, class.	fp::types::product-type
Что такое Sum type? Как вычислить его мощность?	Sum type комбинирует типы через "ИЛИ" — значение является ОДНИМ из вариантов. Мощность = сумма мощностей вариантов. Реализуется как discriminated union с полем <code>_tag</code>. Пример: <code>Circle | Rectangle</code>.	fp::types::sum-type
Что такое ADT (Algebraic Data Type)?	Тип, сконструированный из других типов через Product (И, произведение) и Sum (ИЛИ, сумма). Назван "алгебраическим" потому что операции над типами подчиняются тем же правилам, что и арифметика: <code>never</code> = 0, <code>undefined</code> = 1, <code>|</code> = сложение, <code>&</code> = умножение.	fp::types::adt
Как exhaustive checking работает с типом never?	В default-ветке switch присваиваем значение переменной типа <code>never</code>. Если все варианты sum type обработаны — значение имеет тип <code>never</code> и присвоение корректно. Если вариант пропущен — тип не <code>never</code>, и компилятор выдаёт ошибку.	fp::types::never
Что такое Option и зачем он нужен?	<code>Option&lt;A&gt; = Some&lt;A&gt; | None</code> — sum type для явного представления отсутствия значения. В отличие от <code>string | null</code>, Option заставляет обработать оба случая (Some и None) через map, match или getOrElse. Ошибка ловится при компиляции, а не в runtime.	fp::option
Что делает O.fromNullable в fp-ts?	Конвертирует nullable значение в Option: <code>O.fromNullable(42)</code> → <code>O.some(42)</code>, <code>O.fromNullable(null)</code> → <code>O.none</code>, <code>O.fromNullable(undefined)</code> → <code>O.none</code>. Это мост из мира nullable значений в мир Option.	fp::option::fp-ts
Что делает O.map и чем отличается от прямого вызова функции?	<code>O.map(f)</code> применяет функцию к значению внутри Some, а None пропускает без вызова функции. Позволяет трансформировать значение, не "доставая" его из контейнера и не проверяя, есть ли оно.	fp::option::fp-ts
Что делает O.fromPredicate в fp-ts?	Создаёт "умный конструктор": значение попадает в Some, только если удовлетворяет предикату. <code>O.fromPredicate(n => n > 0)(5)</code> → <code>O.some(5)</code>, <code>O.fromPredicate(n => n > 0)(-3)</code> → <code>O.none</code>.	fp::option::fp-ts
Чем Option отличается от optional chaining (?.) ?	Optional chaining (<code>?.</code>) решает синтаксическую проблему — убирает вложенные if. Но результат имеет тип <code>T | undefined</code>, и вызывающий код может забыть обработать undefined. Option — это отдельный тип, который нельзя использовать как обычное значение без явного извлечения через match/getOrElse.	fp::option
