Что такое Kind (сорт типа) и какие бывают?	Kind — классификация типов по "форме". <code>*</code> — конкретный тип (string, number). <code>* → *</code> — тип-конструктор с 1 параметром (Option, Array, Promise). <code>* → * → *</code> — с 2 параметрами (Either, Map). Option без параметра — не тип, а конструктор типа.	fp::hkt::kind
Почему TypeScript не поддерживает Higher-Kinded Types?	TypeScript generic работает с типами (<code>A</code>, <code>B</code>), но не с тип-конструкторами. Нельзя написать <code>interface Functor&lt;F&gt; { map: (fa: F&lt;A&gt;) => F&lt;B&gt; }</code> — TS не знает, что F принимает параметр. F для него просто тип, а не "шаблон типа".	fp::hkt::limitation
Как fp-ts обходит отсутствие HKT в TypeScript?	Через трюк "Lightweight Higher-Kinded Polymorphism": 1) каждый тип регистрирует строковый URI, 2) глобальный интерфейс URItoKind маппит URI → тип, 3) <code>Kind&lt;URI, A&gt; = URItoKind&lt;A&gt;[URI]</code> делает lookup. Вместо параметризации по тип-конструктору — параметризация по строке.	fp::hkt::uri-trick
Что такое URItoKind в fp-ts?	Глобальный интерфейс-реестр, маппящий строковые URI на конкретные типы. Изначально пустой, типы добавляются через <code>declare module 'fp-ts/HKT' { interface URItoKind&lt;A&gt; { readonly Option: Option&lt;A&gt; } }</code>. Позволяет по строке 'Option' получить тип Option&lt;A&gt;.	fp::hkt::uritokind
Что делает тип Kind в fp-ts?	<code>Kind&lt;URI, A&gt; = URItoKind&lt;A&gt;[URI]</code> — lookup в глобальном реестре. <code>Kind&lt;'Option', number&gt;</code> разрешается в <code>Option&lt;number&gt;</code>. Это позволяет писать обобщённые интерфейсы вроде Functor, параметризованные строкой URI вместо тип-конструктора.	fp::hkt::kind-type
Чем Functor1 отличается от Functor2 в fp-ts?	Суффикс — количество type-параметров тип-конструктора. <code>Functor1&lt;F&gt;</code> — для <code>* → *</code> (Option, Array, Task). <code>Functor2&lt;F&gt;</code> — для <code>* → * → *</code> (Either, Reader, TaskEither). Соответственно используются URItoKind (1 параметр) и URItoKind2 (2 параметра).	fp::hkt::functor1-vs-2
Как зарегистрировать свой тип в fp-ts (3 шага)?	1) Определить тип и строковый URI: <code>const BoxURI = 'Box'; type BoxURI = typeof BoxURI;</code> 2) Зарегистрировать через declare module: <code>declare module 'fp-ts/HKT' { interface URItoKind&lt;A&gt; { readonly Box: Box&lt;A&gt; } }</code> 3) Реализовать инстансы type class: <code>const boxFunctor: Functor1&lt;'Box'&gt; = { URI: 'Box', map: ... }</code>	fp::hkt::register-type
Какие ограничения у URI-трюка fp-ts?	1) Глобальный реестр — declare module загрязняет глобальный scope, два типа с одним URI сломают систему. 2) Нет автоматического вывода инстансов — передаются явно. 3) Boilerplate — для каждого типа нужен URI + declare module + инстансы. Effect (преемник fp-ts) использует другой подход — единый тип Effect&lt;R, E, A&gt;.	fp::hkt::limitations
