## Ad-hoc polymorphism

Ad-hoc polymorphism specifies different behaviors base on the input type.
In ad-hoc polymorphism you need to specify implementation for each type you want to support,
while in parametric you define only one implementation that works for wide range of types.

In ad-hoc polymorphism since this implementation care constant, you can pass them as parameters to functions and return them from functions.
This leads to a way organizing code, known as the **Strategy pattern**

- **Type Class** is a tool in the **Kind Universe**, which allow us to have Ad-hoc Polymorphism

- **Type Class Instance** is an implementation of a **Type Class** for a Type or a Type Constructor

```ts
interface Show<A> {
  toString: (a: A) => string;
}

const numShow: Show<number> = { toString: (a) => a.toString() };
const stringShow: Show<string> = { toString: (a) => a };
const boolShow: Show<boolean> = { toString: (a) => (a ? "Yes" : "No") };
```
