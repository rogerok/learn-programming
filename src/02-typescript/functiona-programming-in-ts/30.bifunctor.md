# Bifunctor

https://www.youtube.com/watch?v=B3iHXFDt_3Y

Bifunctor is a Type class, it knows and specifies a same behavior for a bunch of structures.
`F` represents these kind of structures.
Bifunctor is type of functor that operates on two type parameters

```ts
interface Bifunctor<F> {
  bimap: <A, B, C, D>(
    f: (a: A) => C,
    g: (b: B) => D,
  ) => (fab: HKT2<F, A, B>) => HKT2<F, C, D>;
}
```

Bifunctor needs to follow some laws.

- Identity law states that if we apply identity first and, and then map the objects, it should lead to a same result,
  as if we map the identity arrows first, and then apply the mapped identities.

- Composition law states, if map arrows first and then compose them, it should
  lead to a same result as if we compose the first 2 arrows and then map composed arrow

```
map = (g) => bimap(id, g);
mapLeft = (f) => bimap(f, id);
id = bimap(id, id);
bimap = f => g => mapLeft(f) Ōłś map(g);
bimap = f => g => map(g) Ōłś mapLeft(f);
map (f Ōłś g) = map(f) Ōłś map(g);
mapLeft(f Ōłś g) = mapLeft(f) Ōłś mapLeft(g)
```

## Demo

```ts
import { HKT2, Kind2, HKT3, Kind3, URIS3 } from "fp-ts/HKT";
import { Either, Left, Right, left, right, isLeft, match } from "./lib";

interface Bifunctor<F> {
  URI: F;
  bimap: <A, B, C, D>(
    f: (a: A) => C,
    g: (b: B) => D,
  ) => (fab: HKT2<F, A, B>) => HKT2<F, C, D>;
}

interface Bifunctor2<F extends URIS2> {
  URI: F;
  bimap: <A, B, C, D>(
    f: (a: A) => C,
    g: (b: B) => D,
  ) => (fab: Kind2<F, A, B>) => Kind2<F, C, D>;
}

// interface Bifunctor3<F extends URIS3> ...

const eitherBifunctor: Bifunctor2<"Either"> = {
  URI: "Either",
  bimap: (f, g) =>
    match(
      (a) => left(f(a)),
      (b) => right(g(b)),
    ),
};
```

```ts
type IsEmail = (x: string) => boolean;
const isEmail: isEmail = (x) => x.includes("@");

type ToString = (x: number) => string;
const toString: ToString = (x) => `number ${x}`;

const lift1 = eitherBifunctor.bimap(isEmail, toString);
console.log(lift1(left("bob@email.com"))); // {_tag: "Left", left: true};
console.log(lift1(right(12))); // {_tag: "Right", right: "number 12"};

type Id = <A>(x: A) => A;
const id: Id = (x) => x;

const lift2 = eitherBifunctor.bimap(id, toString);
lift2(left("bob@email.com")); // {_tag: "Left", left: "bob@email.com"}
lift2(right(12)); // {_tag: "Right", right: "number 12"};
```

Bifunctor is a functor that works with the structures with at least 2 type parameters

```ts
const lift3 = eitherBifunctor.bimap(isEmail, id);
lift3(left("bob@email.com")); // {_tag: "Left", left: true}
lift3(right(12)); // {_tag: "Right", right: 12}
```

`lift3` is converting the left values and not the right ones
Since we could map over `Left` and `Right` of `Either` type parameters, we say `Either` is functorial on both of its type parameters.
As a matter of fact we can go and rename `lift2` to `map`, and `lift3` to `mapLeft`

```ts
const map = eitherBifunctor.bimap(id, toString);
map(left("bob@email.com")); // {_tag: "Left", left: "bob@email.com"}
map(right(12)); // {_tag: "Right", right: "number 12"};

const mapLeft = eitherBifunctor.bimap(isEmail, id);
mapLeft(left("bob@email.com")); // {_tag: "Left", left: true}
mapLeft(right(12)); // {_tag: "Right", right: 12}
```

So map maps over the `Right` type parameter of `Either` and `mapLeft` maps over the `Left` type parameter of `Either`.

```ts
const lift4 = eitherBifunctor.bimap(id, id);
lift4(left("bob@email.com")); // {_tag: "Left", left: "bob@email.com"}
lift4(right(12)); // {_tag: "Right", right: 12}
```
