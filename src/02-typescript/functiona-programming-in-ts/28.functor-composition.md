## Demo

```ts
type Double = (x: number) => number;
const double: Double = (x) => x * 2;

const data1: Option<number> = some(12);

const output1 = optionFunctor.map(double)(data1);

console.log(output1); // {_tag: "Some", value: 24}

const data2 = some(cons(1, cons(2, cons(3, nil))));
```

How can we reach inside data2's structure and double numbers inside it?

First of all, lets define type of data2

```ts
const data2: Option<List<number>> = some(cons(1, cons(2, cons(3, nil))));

const mapO = optionFunctor.map;
const mapL = listFunctor.map;

// const output2 = mapO((xs: List<number>) => mapL(double)(xs))(data2);

// We can rewrite output2 to point free form
const output_2 = mapO(mapL(double))(data2);
```

The output of `mapL` function passed as input of `mapO` function. So this is function composition of `mapO` and `mapL`

Lets rewrite it

```ts
const output2 = compose(mapO, mapL)(double)(data2); // Typescript throws an errorr
```

But we can fix it

```ts
type ComposeR = <A, B, C>(f: (a: A) => B, g: (b: B) => C) => (a: A) => C;
const composeR: ComposeR = (f, g) => (a) => g(f(a));

const output2 = composeR(mapL, mapO)(double)(data2);
```

Next lets try to define a Functor instance for
our structure Option of List by reusing existing Functor instances for Option and List.

```ts
declare module "fp-ts/HKT" {
  interface URItoKind<A> {
    OptionList: Option<List<A>>;
  }
}

const optionListFunctor: Functor1<"OptionList"> = {
  URI: "OptionList",
  map: composeR(mapL, mapO),
};

const output3 = optionListFunctor.map(double)(data2);

console.log(output3);
```
