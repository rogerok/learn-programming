# Task and Asynchronous Side Effects

Task is asynchronous IO

- Task models computation
- Task has side effects
- Task has no exceptions
- Task is Asynchronous
- Task never fails

```ts
type Task<A> = IO<Promise<A>>;
```

## Error handling

```ts
Task<Either<E, A>>;
```

- Task abstracts promise
- Task protects us from missing promise rejections
- Task is an object and needs to triggered to execute
- Code written in Task looks similar to code written with IO
- Task and IO allow us to abstract Synchronous and Asynchronous concerns

```ts
function rollDice(): IO<number> {
  return of(Math.ceil(Math.random() * 6));
}
```

```ts
const rolledDice: () => IO<number> = () => of(Math.ceil(Math.random() * 6));
```

```ts
const rolledDice: () => Task<number> = () =>
  of(MathServer.random()).then((x) => Math.ceil(x * 6));
```

```ts
const luckyIO = rolledDice.map((x) => x % 2 === 0);
// IO<Boolean>     // IO<number>
```

```ts
const luckyTask = rolledDice.map((x) => x % 2 === 0);
// Task<Boolean>     // Task<number>
```
