# Бивариантность методов в объектах

## Метод vs Свойство-функция

TypeScript различает два способа записи функции в объектном типе, и они ведут себя **по-разному**:

```ts
interface Entity { id: string }
interface User extends Entity { name: string }
interface CompanyUser extends User { company: string }
```

### Запись как метод (бивариантна)

```ts
type Processor1 = {
  name: string;
  handler(e: User): User; // ← метод
};

const processor1: Processor1 = {
  name: "logger",
  handler(u: CompanyUser) { // OK — принимает более узкий тип
    console.log(u);
    // return {} as CompanyUser; // ❌ ошибка только если возврат уже
  },
};
```

Параметр `handler` **бивариантен**: допускается как более узкий (`CompanyUser`), так и более широкий (`Entity`) тип аргумента.

> Бивариантность по параметрам — намеренная особенность для обратной совместимости.

**Но:** бивариантность распространяется **только на входящие аргументы**. Возвращаемое значение всё равно проверяется ковариантно — вернуть `CompanyUser` вместо `User` допустимо (подтип), но вернуть `Entity` — нет.

### Запись как свойство-функция (контравариантна по параметрам)

```ts
type Processor2 = {
  name: string;
  handler: (e: User) => void; // ← свойство-функция
};

const processor2: Processor2 = {
  name: "logger",
  handler: (u: CompanyUser) => { // ❌ Error — более узкий тип не допускается
    console.log(u);
  },
};
```

Со `strictFunctionTypes` свойства-функции проверяются **контравариантно**: параметры должны быть такими же или шире ожидаемого.

---

## Почему это важно

| Запись | Вариантность параметра | Безопасна? |
|---|---|---|
| `handler(e: User): User` | бивариантна | ❌ потенциально опасна |
| `handler: (e: User) => User` | контравариантна | ✅ безопасна |

**Рекомендация:** для публичных API используйте запись свойства-функции (`handler: (e: T) => R`) — она даёт более строгую проверку.

---

## Почему бивариантность методов существует

TypeScript ввёл бивариантность методов до появления `strictFunctionTypes` для совместимости с паттернами вроде:

```ts
interface Array<T> {
  forEach(callback: (value: T, index: number) => void): void;
}
```

Если бы методы были контравариантны по параметрам, многие распространённые паттерны перестали бы компилироваться.
