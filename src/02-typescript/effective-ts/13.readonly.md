# Readonly

## `Readonly<T>` — поверхностная защита

`Readonly<T>` запрещает переприсваивание свойств объекта **по месту использования**, но не является глубоким:

```ts
interface User {
  id: string;
  name: string;
}

const user: Readonly<User> = {
  id: "id",
  name: "John",
};

user.id = "new"; // ❌ Cannot assign to 'id' because it is a read-only property
```

### Readonly не защищает при передаче в функцию

```ts
const process = (u: User) => {
  u.id = "123"; // Нет ошибки — параметр типа User, не Readonly<User>
};

process(user); // Ошибки не возникает!
```

`Readonly` контролирует **только то место, где он указан**. В сигнатуре `process` идентификатор `u` имеет тип `User` (без `Readonly`) — поэтому запись разрешена.

## `Readonly` — shallow (поверхностный)

Вложенные объекты остаются **мутабельными**:

```ts
interface Config {
  db: { host: string; port: number };
}

const config: Readonly<Config> = {
  db: { host: "localhost", port: 5432 },
};

config.db = { host: "other", port: 0 }; // ❌ Error — нельзя переприсвоить db
config.db.port = 9999;                  // OK! — вложенный объект не защищён
```

То же касается `readonly` в массивах:

```ts
const arr: readonly number[] = [1, 2, 3];
arr.push(4);  // ❌ Error
arr[0] = 99;  // ❌ Error
```

## `as const` — глубокий Readonly

`as const` применяет `readonly` **рекурсивно** и сужает типы до литералов:

```ts
const config = {
  a: 1,
  b: 2,
  c: {
    e: 3,
  },
} as const;
// Тип: { readonly a: 1; readonly b: 2; readonly c: { readonly e: 3 } }

config.c.e = 10; // ❌ Error — защита работает на всех уровнях
```

---

## Сравнение

| | `Readonly<T>` | `as const` |
|---|---|---|
| Глубина | Поверхностная (1 уровень) | Глубокая (рекурсивная) |
| Тип полей | Сохраняет оригинальные типы | Сужает до литеральных типов |
| Применение | Аннотация типа | Выражение на значении |
| Работает с вложенными | ❌ | ✅ |

---

## `ReadonlyArray<T>` vs `readonly T[]`

Оба синтаксиса эквивалентны и запрещают мутацию:

```ts
const a: ReadonlyArray<number> = [1, 2, 3];
const b: readonly number[]     = [1, 2, 3];

// Совместимость:
const mutable: number[] = [1, 2, 3];
const immutable: readonly number[] = mutable; // OK — mutable → readonly безопасно

const back: number[] = immutable; // ❌ — readonly → mutable запрещено
```
