# TypeScript — Иерархия типов и проверки

## Иерархия типов

В TypeScript существует **иерархия типов**, где одни типы являются более **узкими**, а другие **более широкими**.

Общий принцип:

> **Более узкий тип можно присвоить более широкому, но не наоборот.**

### Порядок (упрощённо)

```
never
  ↓
Literal Types
  ↓
Primitive Types (string, number, boolean)
  ↓
Union Types
  ↓
unknown /  - самый верхний тип в иерархии типов. включает любые значения.
any - any отключает вывод типов
```

---

## `never`

`never` находится **в самом низу иерархии типов**.

Причина:  
`never` представляет **пустое множество значений**.

В теории множеств:

> Пустое множество является подмножеством любого множества.

Поэтому:

```
never → можно присвоить любому типу
```

Пример:

```ts
let x: never;

let s: string = x;
let n: number = x;
```

---

## Literal Types

Literal types — это **максимально узкие типы**, представляющие **конкретное значение**.

Примеры:

```ts
type A = 42;
type B = "hello";
type C = true;
```

Пример присваивания:

```ts
const first: 3 = 3;
```

Но:

```ts
const second: 3 = 2 + 1;
// ❌ Type 'number' is not assignable to type '3'
```

Почему?

Потому что `2 + 1` имеет тип `number`, а не literal `3`.

---

## Union Types

Union type объединяет несколько типов.

```ts
type Bool = true | false;
```

Union **шире**, чем каждый отдельный literal.

```
true ⊂ true | false
```

Поэтому:

```
literal → union (можно)
union → literal (нельзя)
```

---

## Проверка типов через `extends`

TypeScript использует **условные типы**, которые проверяют:

> Является ли один тип подтипом другого.

```ts
type Foo = 42 extends number ? "YES" : "NO";
```

Результат:

```
YES
```

Потому что:

```
42 ⊂ number
```

Другой пример:

```ts
type Foo2 = number extends 42 ? "YES" : "NO";
```

Результат:

```
NO
```

Потому что:

```
number ⊄ 42
```

# Excess Property Check

**Excess property check** — это дополнительная проверка TypeScript, которая помогает ловить **опечатки в object literal
**.

Пример:

```ts
type Foo = {
  foo: string;
};

const s: Foo = {
  foo: "hello",
  a: 3,
};
```

Ошибка:

```
Object literal may only specify known properties
```

Причина:

TypeScript предполагает, что `a` — **опечатка**.

---

## Почему ошибка появляется только у object literal

TypeScript выполняет **excess property check только для object literal**.

Если объект сначала записан в переменную — проверка отключается.

Пример:

```ts
type Foo = {
  foo: string;
  bar?: number;
};

const obj = {
  foo: "hello",
  baaar: 3,
};

const s: Foo = obj;
```

Ошибки **нет**.

Почему?

TypeScript проверяет **структурную совместимость типов**, а не точное совпадение.

Тип `obj`:

```
{
  foo: string
  baaar: number
}
```

Тип `Foo` требует:

```
{
  foo: string
  bar?: number
}
```

Условия выполняются:

- есть обязательное поле `foo`
- `bar` опциональное

Поэтому типы считаются **совместимыми**.
