# Generics and Overloads

## Generics — переменные для типов

Дженерики позволяют параметризировать тип, не теряя информацию о нём.

```ts
function identity<T>(value: T): T {
  return value;
}

identity(42)       // T = 42
identity("hello")  // T = "hello"
```

---

## Type Constraints (`extends`)

`extends` в дженерике описывает **ограничение** — какие операции допустимы внутри функции.

```ts
function pick<Obj extends object, Key extends keyof Obj>(obj: Obj, key: Key) {
  return obj[key]; // TypeScript знает, что key — ключ obj
}
```

Без `extends object` TypeScript не разрешил бы `keyof Obj`.

---

## Вывод дженерика в функциях

TypeScript выводит дженерик **только из объявления параметров**, не из тела функции:

```ts
function wrap<T>(value: T): { value: T } {
  return { value };
}

wrap(42)  // T = 42, возвращает { value: 42 }
```

---

## `infer` — извлечение типа

`infer` позволяет **захватить** часть типа в переменную внутри conditional type.

```ts
type Ref<Value> = { value: Value }
type MaybeRefOrCallable<Value> = Value | Ref<Value> | (() => Value)
```

Без `infer` извлечь `Value` из `MaybeRefOrCallable<...>` сложно. С `infer`:

```ts
type ExtractValue<T extends MaybeRefOrCallable<unknown>> =
  T extends MaybeRefOrCallable<infer Value> ? Value : never;

type A = ExtractValue<Ref<string>>          // string
type B = ExtractValue<() => number>         // number
type C = ExtractValue<boolean>              // boolean
```

`infer Value` говорит: "если `T` совместим с `MaybeRefOrCallable<...>`, сохрани первый аргумент в `Value`".

---

## Встроенные utility types на основе `infer`

```ts
type ReturnType<T extends (...args: any) => any> =
  T extends (...args: any) => infer R ? R : any;

type Parameters<T extends (...args: any) => any> =
  T extends (...args: infer P) => any ? P : never;
```

## Function Overloads

Перегрузки позволяют задать **несколько сигнатур** для одной функции.

```ts
function format(value: string): string;
function format(value: number): string;
function format(value: string | number): string {
  return String(value);
}
```

Правила:
- Реализация должна быть **совместима** со всеми перегрузками.
- Сигнатура реализации **не видна** снаружи — только объявленные перегрузки.

### Когда использовать перегрузки

Перегрузки нужны, когда тип возвращаемого значения **зависит от типа входного**:

```ts
function parse(input: string): number;
function parse(input: number): string;
function parse(input: string | number): string | number {
  return typeof input === 'string' ? parseInt(input) : String(input);
}

const a = parse("42");  // тип: number (не string | number!)
const b = parse(42);    // тип: string
```

Без перегрузок `parse("42")` вернул бы `string | number`.
