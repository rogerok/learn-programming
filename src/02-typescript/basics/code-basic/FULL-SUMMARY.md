## 1. Система типов

**Тип** — множество допустимых значений + набор операций над ними.

**Type system** — правила назначения типов конструкциям программы. Задачи: safety, абстракция, документация, оптимизация.

TypeScript использует **структурную типизацию** — совместимость по форме, а не по имени. TypeScript **не является sound** — допускает некоторые небезопасные операции (`any`, ковариантные массивы, бивариантные методы) ради практичности.

---

## 2. Типы как множества

| Тип | Множество |
|---|---|
| `never` | ∅ (пустое) |
| `'hello'` | { "hello" } |
| `boolean` | { true, false } |
| `number` | все числа |
| `string` | все строки |
| `unknown` | универсальное множество |

### Операции

- **Union `|`** — объединение множеств. `string | number` = все строки + все числа.
- **Intersection `&`** — пересечение. Для объектных типов: объединяет поля. Для примитивов: `string & number` → `never`.
- **Subtyping** — `A ⊂ B`: все значения A являются значениями B.

Чем **меньше** множество — тем **у́же** (конкретнее) тип. Чем **больше** — тем **шире**.

---

## 3. Иерархия типов

```
never                       — bottom type (∅)
  ↓
Literal Types (42, "hello") — одноэлементные множества
  ↓
Primitives (number, string) — бесконечные множества
  ↓
Union Types
  ↓
unknown                     — top type
any                         — вне системы типов
```

**Подтип** присваивается **супертипу**, но не наоборот.

### any vs unknown

| | `any` | `unknown` |
|---|---|---|
| Принимает всё | Да | Да |
| Присваивается всему | Да | Нет (только `unknown`/`any`) |
| Операции | Любые | Запрещены до сужения |

`unknown` — безопасная альтернатива `any`: заставляет проверить тип перед использованием.

## 4. Literal Types

Тип, представляющий одно конкретное значение. Доступны для `string`, `number`, `boolean`, `bigint`.

```ts
const x = 'hello';  // тип "hello" — const выводит литерал
let y = 'hello';    // тип string  — let расширяет
```

### `as const`

Сужает до литерального типа + `readonly`:

```ts
const config = { type: 'mysql', port: 5432 } as const;
// { readonly type: "mysql"; readonly port: 5432 }
```

Для массивов → readonly tuple. Для простых значений → литеральный тип.

### Template Literal Types

```ts
type EventName = 'click' | 'scroll';
type Handler = `on${Capitalize<EventName>}`; // "onClick" | "onScroll"
```

---

## 5. never

**Пустое множество** — bottom type. Функция возвращает `never`, если гарантированно не завершается (throw, бесконечный цикл).

| Свойство | Следствие |
|---|---|
| `T \| never` = `T` | Нейтральный элемент union |
| `T & never` = `never` | Поглощает в intersection |
| `never` ⊂ любой тип | Присваивается чему угодно |

Используется для **фильтрации** в distributive conditional types и **exhaustive check**.

---

## 6. void

`void ≠ undefined`. Два контекста:

| Контекст | Поведение |
|---|---|
| Тип колбека (`type Fn = () => void`) | Разрешает любой возврат, результат игнорируется |
| Явная аннотация (`function f(): void`) | Запрещает `return <value>` |

Причина: `forEach` колбек типизирован как `() => void`, чтобы `result.push(n)` (возвращает `number`) не давал ошибку.

---

## 7. Assignability (Присваиваемость)

Значение типа `A` присваивается типу `B`, если `A ⊂ B` (A — подмножество B).

Для объектов: больше полей → у́же тип → подмножество типа с меньшим числом полей.

**Excess Property Check** — дополнительная проверка **только для object literals**: запрещает лишние поля (ловит опечатки).

---

## 8. Структурная типизация

Совместимость определяется **формой** (наличием нужных полей), а не именем типа.

- Объект с лишними полями совместим с типом без этих полей.
- Исключение: классы с `private`/`protected` полями проверяются **номинально**.

---

## 9. Ковариантность и контравариантность

```
Параметры функции — контравариантны (ожидаемые ≥ фактические по ширине)
Возврат функции   — ковариантен    (фактический ≤ ожидаемого по ширине)
```

- `strictFunctionTypes` включает контравариантную проверку параметров.
- Методы объектов (`method(): void`) всегда бивариантны — для строгости используйте `method: () => void`.
- Массивы ковариантны (технически небезопасно для мутабельных); `readonly` массивы — безопасно ковариантны.

---

## 10. Type Refinement (Сужение типов)

TypeScript анализирует control flow и отсекает невозможные варианты union.

**Безопасные**: `typeof`, `instanceof`, дискриминатор, литеральное сравнение, type guard.
**Небезопасные**: `in` (структурная типизация допускает лишние поля).

### Discriminated Unions

```ts
type Event =
  | { type: 'login'; user: string }
  | { type: 'logout'; timestamp: number };
```

Литеральное поле-дискриминатор позволяет безопасно сужать тип.

### Exhaustive Check

`assertNever(x: never): never` — гарантирует обработку всех вариантов union. При добавлении нового варианта — ошибка компиляции.

---

## 11. Intersection Types (`&`)

Для объектов — объединяет поля. Для примитивов — `never` (нет общих значений).

Для объектных типов интуиция инвертирована:
- `A | B` → больше значений, но доступны только общие поля
- `A & B` → меньше значений, но доступны все поля обоих типов

Конфликт одноимённых несовместимых свойств → поле становится `never`.

---

## 12. Tuples

Массив **фиксированной длины** с **типизированными позициями**.

```ts
const point: [number, number] = [1, 3];
```

- `push`/`pop` не запрещены (обратная совместимость) — используйте `readonly` для защиты.
- Именованные элементы: `[start: number, end: number]` (TS 4.0+).
- Rest-элементы: `[string, ...number[]]`.
- `as const` превращает массив в readonly tuple с литеральными типами.

---

## 13. Readonly Arrays

`readonly number[]` (или `ReadonlyArray<number>`) запрещает мутирующие операции (`push`, `sort`, индексная запись).

- `readonly` **не рекурсивен** — вложенные объекты остаются мутабельными.
- `number[]` → `readonly number[]` (OK, безопасно). Обратное — ошибка.

---

## 14. Типы object / Object / {}

| Тип | Что принимает |
|---|---|
| `{}` | Всё кроме `null`/`undefined` (не описывает пустой объект!) |
| `Object` | Как `{}`, но предопределяет `toString`, `valueOf` и др. |
| `object` | Только непримитивы |

На практике: `object` для непримитивов, `Record<string, unknown>` для произвольных объектов, конкретный тип — когда знаете структуру.

## 15. Интерфейсы

### interface vs type

| | `interface` | `type` |
|---|---|---|
| Declaration merging | Да | Нет |
| Union/Intersection | Нет | Да |
| Mapped/conditional types | Нет | Да |

### `implements` не наследует типы

TypeScript **только проверяет** соответствие — типы параметров нужно писать вручную.

### Опциональные свойства интерфейса не попадают в класс автоматически.

## 16. Классы как типы

Без приватных полей — чисто структурная совместимость. С `private`/`protected` — номинальная (требуется экземпляр класса).

ES2022 `#`-поля дают настоящую рантайм-приватность (в отличие от TS `private`, стираемого при компиляции).

---

## 17. Модификаторы доступа

| Модификатор | Доступ |
|---|---|
| `public` | Отовсюду |
| `private` | Только класс |
| `protected` | Класс + наследники |

**Parameter Properties** — модификатор в параметре конструктора создаёт и инициализирует поле:

```ts
class User {
  constructor(public name: string, private age: number) {}
}
```

---

## 18. Статические члены

Принадлежат классу, а не экземпляру. Поддерживают модификаторы доступа. Наследуются подклассами. `static {}` блоки (ES2022) для сложной инициализации.

---

## 19. Assert-функции

`asserts value is T` — рантайм-проверка + сужение типа. В отличие от type guard (`is` → boolean), assert бросает ошибку при неудаче.

- Type Guard → expression-level narrowing (внутри `if`)
- Assert → statement-level narrowing (после вызова)

TypeScript **доверяет** логике assert-функции — ответственность на разработчике.

---

## 20. Mapping Modifiers

В mapped types: `+`/`-` добавляют/удаляют `readonly` и `?`:

```ts
type Required<T> = { [P in keyof T]-?: T[P] };
type Mutable<T> = { -readonly [P in keyof T]: T[P] };
```

Key Remapping (TS 4.1+): `as` в mapped type для переименования/фильтрации ключей.

Гомоморфные mapped types (`keyof T`) сохраняют модификаторы оригинала.

---

## 21. Функции как параметры

Тип `Function` отключает проверку → всегда `any`. Используйте явные сигнатуры:

```ts
function process(callback: (input: string) => number) {}
```

`(...args: any[]) => any` — типобезопаснее `Function` для «любой функции».

---

## 22. Generics

Переменные для типов. `extends` — ограничение (constraint). TypeScript выводит generics из аргументов вызова.

### `infer`

Извлечение части типа внутри conditional types:

```ts
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;
```

---

## 23. Spread-оператор

`[1, 2]` выводится как `number[]`, а не `[number, number]`. Для spread в функцию с фиксированными параметрами: `as const` или явная аннотация кортежа.

Spread объектов: последнее свойство wins при конфликте имён.

---

## 24. Приведение типов

- **Upcasting** (восходящее): подтип → супертип. Безопасно, автоматически.
- **Downcasting** (нисходящее) через `as`: небезопасно — компилятор доверяет программисту.
- `as const`: нисходящее к литеральному типу + readonly.

---

## 25. Enum vs Union of Literals

Enum — рантайм-объект (остаётся в JS). Union of literals — стирается при компиляции (zero runtime cost). Многие предпочитают union: нет артефакта, проще tree-shaking.
