# Возврат в асинхронном коде

В прошлой главе мы выяснили, что асинхронные ф-ции никогда не возвращают результат асинхронной операции.

```ts
import fs from "fs";

const noop = () => {};
const content = fs.readFile("./myfile", "utf-8", noop);
console.log(content); // undefined
```

Единственный способ получить результат - описать логику в коллбеке. Возникает вопрос: а что если сделать return внутри коллбека?

```ts
import fs from "fs";

const content = fs.readFile("./myfile", "utf-8", (_error, data) => {
  // что-нибудь делаем
  return data;
});
console.log(content); // undefined
```

Ничего не меняется, т.к. возрат никем не используется, но инструкция `return` не бесполезна, ее можно использовать как способ прервать асинхронный кож.

```ts
import fs from "fs";

const content = fs.readFile("./myfile", "utf-8", (_error, data) => {
  if (data === "") {
    return;
  }
  // делаем что-нибудь с данными
});
console.log(content); // undefined
```

Напишем асинхронную ф-цию обертку.
Сразу вспоминаем - если наша ф-ция асинхронная, то она должна иметь общепринятую сигнатуру т.е. принимать первым параметром ошибку и вторым сами данные.
Возврата данных через return быть не может.

```ts
import fs from "fs";

const readWileWithTrim = (filepath, cb) => {
  fs.readFile(filepath, "utf-8", (err, data) => {
    // При вызове колбек-функции передаем null первым параметром
    // Поскольку обработку ошибок мы пока не рассматриваем
    cb(null, data.trim());
  });
};

readFileWithTrim("./myfile", (_error, data) => console.log(data));
```

Этот процесс рекурсивен по своей природе, любая функция, которая внутри работает с асинхронной функцией, становится асинхронной и начинает принимать на вход колбек. Почему так происходит? Почему нельзя просто выполнить асинхронную операцию внутри, никак не сообщая об этом наружу? Дело в том, что в такой ситуации вы не можете ни воспользоваться результатом работы асинхронной функции (ведь данные приходят в колбек в другом стеке вызовов), ни узнать о том, закончилась ли операция вообще и закончилась ли она успешно.
