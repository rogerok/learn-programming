Что такое байт и сколько значений он может хранить?	Байт = 8 бит = наименьшая адресуемая единица памяти. Хранит значения 0..255 (256 вариантов, 0x00–0xFF).	csapp::2.1::basics
Hex: сколько бит кодирует одна hex-цифра? Пример с CSS-цветом.	Одна hex-цифра = ровно 4 бита. Пример: #FF8800 — три байта: FF=255 (red), 88=136 (green), 00=0 (blue).	csapp::2.1::hex
Endianness: как число 0x01234567 хранится по адресу 0x100 в little-endian?	0x100: 67, 0x101: 45, 0x102: 23, 0x103: 01. Младший байт (LSB = 0x67) по наименьшему адресу. Аналогия: европейская дата — маленькое первым (ДД.ММ.ГГГГ).	csapp::2.1::endianness
Назови три практических случая, когда endianness важен.	1) Сетевые протоколы (TCP/IP = big-endian). 2) Дизассемблер (байты инструкций в порядке платформы). 3) Type punning (чтение float как u32 — порядок байтов определяет результат).	csapp::2.1::endianness
Как в Zig посмотреть на сырые байты переменной?	std.mem.asBytes(&x) — возвращает слайс []u8, показывающий байтовое представление x в памяти.	csapp::2.1::zig
Что возвращает @sizeOf(T) в Zig? Чем usize отличается от u64?	@sizeOf(T) — количество байтов типа T (comptime). u64 всегда 8 байт. usize = размер указателя: 4 байта на 32-bit, 8 на 64-bit. usize используется для индексов и адресов.	csapp::2.1::zig
Чем отличается ~ от ! в Zig?	~ — побитовое NOT для целых чисел (инвертирует каждый бит). ! — логическое NOT только для bool. Применение ! к числу — ошибка компиляции.	csapp::2.1::zig::bitwise
Зачем писать ~@as(u32, 0xFF) а не ~0xFF в Zig?	Литерал 0xFF имеет тип comptime_int (бесконечная точность). ~0xFF = -256, а не 0xFFFFFF00. @as(u32, 0xFF) фиксирует ширину 32 бита, и ~ даёт правильное 0xFFFFFF00.	csapp::2.1::zig::bitwise
Как извлечь красный канал из RGB-цвета 0xFF8800 через сдвиги и маски?	(color >> 16) & 0xFF = 0xFF = 255. Сдвиг вправо на 16 бит перемещает красный канал в младший байт, маска 0xFF оставляет только его.	csapp::2.1::bitwise::rgb
Как через побитовые операции проверить/добавить/убрать Unix permission?	Проверить execute у owner: perms & 0o100 != 0. Добавить write для group: perms | 0o020. Убрать write для others: perms & ~@as(u9, 0o002).	csapp::2.1::bitwise::permissions
Побитовые vs логические: чему равны 0x66 & 0x39 и (0x66 != 0) and (0x39 != 0)?	0x66 & 0x39 = 0x20 (побитовая обработка каждой пары бит). (0x66 != 0) and (0x39 != 0) = true (оба ненулевые). Побитовая даёт число, логическая — bool.	csapp::2.1::bitwise::logical
Что такое short-circuit evaluation? Пример.	Ленивое вычисление: false and expr — expr не вычисляется. Пример: b != 0 and @divTrunc(a, b) > 5 — если b == 0, деление не выполняется.	csapp::2.1::logical
Чем >> отличается от >>> в JavaScript? Аналог в Zig?	>> — арифметический (копирует знаковый бит). >>> — логический (заполняет нулями). (-1 >>> 0) = 4294967295. В Zig тип определяет поведение: u8 >> 2 = логический, i8 >> 2 = арифметический.	csapp::2.1::shifts
В чём разница между арифметическим и логическим правым сдвигом?	Логический (unsigned): слева заполняется нулями. Арифметический (signed): слева копируется знаковый бит, сохраняя знак числа. Пример: i8(-76) >> 2 = -19 (единицы слева), u8(180) >> 2 = 45 (нули слева).	csapp::2.1::shifts
Log2Int: почему правый операнд сдвига для u8 имеет тип u3?	Макс. допустимый сдвиг для u8 — 7 (на 8 уже нельзя). Числа 0–7 вмещаются в u3 (log2(8) = 3). Zig требует тип Log2Int(T) — невозможно даже выразить недопустимый сдвиг.	csapp::2.1::shifts::zig
Null-terminator в C/Zig vs длина строки в JS: в чём разница?	C/Zig: строка заканчивается байтом 0x00 (null-terminator). Zig: *const [N:0]u8, .len = байты. JS: "str".length = UTF-16 code units (не байты!). "Привет".length = 6, но реальных UTF-8 байт = 12.	csapp::2.1::strings
Чему равно "Привет".length в JS и "Привет".len в Zig?	JS: .length = 6 (UTF-16 code units, каждая кириллическая буква = 1 code unit). Zig: .len = 12 (UTF-8 байты, каждая кириллическая буква = 2 байта). Реальный размер в памяти ближе к Zig-ответу.	csapp::2.1::strings::encoding
