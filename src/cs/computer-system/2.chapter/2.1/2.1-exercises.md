## Упражнение 1: show_bytes

**Сложность:** начальный уровень

### Условие

В книге функция `show_bytes` демонстрирует, как многобайтное значение хранится в памяти. Реализуй её аналог на Zig.

Напиши функцию `showBytes`, которая принимает метку (строку) и срез байтов `[]const u8`, и выводит каждый байт в hex-формате через пробел.

Затем в `main` проверь:
- `i32 = 12345` даёт байты `39 30 00 00` (little-endian x86)
- `f32 = 12345.0` даёт байты `00 e4 40 46`

Для тестовой части: напиши тест, который проверяет байтовое представление `i32 = 12345` с помощью `std.testing.expectEqualSlices`.

**Требования:**
- Использовать `std.mem.asBytes` для получения байтов
- Вывод формата: `label: XX XX XX XX`
- Тест: `std.testing.expectEqualSlices(u8, expected, actual)`

> [!tip]- Подсказка 1
> `std.mem.asBytes(&value)` возвращает `*[N]u8` — указатель на массив байтов. Чтобы сравнить его со срезом в тесте, используй `&` для взятия адреса и убедись, что типы совпадают.

> [!tip]- Подсказка 2
> Для форматированного вывода hex-байта в `std.debug.print` используй спецификатор `{x:0>2}` — это значит: hex, минимум 2 символа, дополнить нулями слева.

> [!tip]- Подсказка 3
> В тесте `expected` — это литерал массива: `&[_]u8{ 0x39, 0x30, 0x00, 0x00 }`. `actual` — это результат `std.mem.asBytes(&val)`, который нужно привести к `[]const u8`.

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
>
> // Выводит байты значения в hex-формате
> fn showBytes(label: []const u8, bytes: []const u8) void {
>     std.debug.print("{s}: ", .{label});
>     for (bytes) |b| {
>         std.debug.print("{x:0>2} ", .{b});
>     }
>     std.debug.print("\n", .{});
> }
>
> pub fn main() void {
>     const i: i32 = 12345;
>     const f: f32 = 12345.0;
>
>     // std.mem.asBytes возвращает *[N]u8 — указатель на байтовое представление
>     showBytes("i32 12345  ", std.mem.asBytes(&i));
>     showBytes("f32 12345.0", std.mem.asBytes(&f));
>
>     // Ожидаемый вывод на little-endian (x86):
>     // i32 12345  : 39 30 00 00
>     // f32 12345.0: 00 e4 40 46
> }
>
> test "байтовое представление i32 = 12345" {
>     const val: i32 = 12345;
>     const bytes = std.mem.asBytes(&val);
>
>     // 12345 = 0x00003039
>     // В little-endian младший байт идёт первым: 39 30 00 00
>     const expected = &[_]u8{ 0x39, 0x30, 0x00, 0x00 };
>     try std.testing.expectEqualSlices(u8, expected, bytes);
> }
>
> test "байтовое представление f32 = 12345.0" {
>     const val: f32 = 12345.0;
>     const bytes = std.mem.asBytes(&val);
>
>     // IEEE 754: 12345.0 = 0x4640E400
>     // В little-endian: 00 e4 40 46
>     const expected = &[_]u8{ 0x00, 0xe4, 0x40, 0x46 };
>     try std.testing.expectEqualSlices(u8, expected, bytes);
> }
> ```

### Разбор

`std.mem.asBytes` — ключевой инструмент этого упражнения. Он делает то же самое, что в C делает каст `(unsigned char *)&val`: смотрит на байты значения напрямую, без преобразования.

На x86 (little-endian) `12345 = 0x00003039` хранится как `39 30 00 00` — **младший** байт по **меньшему** адресу. Именно поэтому значение "читается наоборот" при просмотре побайтно.

Float `12345.0` имеет то же числовое значение, но совершенно другое битовое представление — `0x4640E400` по стандарту IEEE 754. Это иллюстрирует главную идею раздела: одни и те же байты, разная интерпретация.

## Упражнение 2: Определение эндианности

**Сложность:** начальный уровень

### Условие

Напиши функцию `isLittleEndian() bool`, которая во время выполнения определяет порядок байтов текущей машины.

Два способа на выбор (реализуй оба и убедись, что результат совпадает):
1. Через встроенный тип `@import("builtin").target.cpu.arch.endian()`
2. Через запись `u16 = 1` в память и проверку первого байта

Напиши тест, который проверяет что твоя машина действительно little-endian.

**Требования:**
- Оба варианта должны возвращать одинаковый результат
- Тест использует `std.testing.expect`

> [!tip]- Подсказка 1
> Запись `u16 = 1` в little-endian хранится как `01 00`. Первый байт (`[0]`) будет равен `1`. В big-endian — наоборот: `00 01`, первый байт будет `0`.

> [!tip]- Подсказка 2
> `std.mem.asBytes(&val)[0]` — так можно получить первый байт значения `val`.

> [!tip]- Подсказка 3
> `@import("builtin").target.cpu.arch.endian()` возвращает значение типа `std.builtin.Endian` — это перечисление с вариантами `.little` и `.big`.

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
> const builtin = @import("builtin");
>
> // Способ 1: через встроенный тип builtin
> fn isLittleEndianComptime() bool {
>     return builtin.target.cpu.arch.endian() == .little;
> }
>
> // Способ 2: через запись значения в память и проверку байта
> fn isLittleEndianRuntime() bool {
>     const val: u16 = 1;
>     // В little-endian: байты "01 00", первый байт == 1
>     // В big-endian:    байты "00 01", первый байт == 0
>     return std.mem.asBytes(&val)[0] == 1;
> }
>
> pub fn main() void {
>     const via_builtin = isLittleEndianComptime();
>     const via_memory  = isLittleEndianRuntime();
>
>     std.debug.print("Через builtin: {}\n", .{via_builtin});
>     std.debug.print("Через память:  {}\n", .{via_memory});
>     std.debug.print("Совпадают: {}\n",     .{via_builtin == via_memory});
> }
>
> test "оба метода дают одинаковый результат" {
>     try std.testing.expectEqual(isLittleEndianComptime(), isLittleEndianRuntime());
> }
>
> test "x86 — little-endian" {
>     // Этот тест пройдёт только на little-endian машине (x86/x86-64/ARM)
>     try std.testing.expect(isLittleEndianRuntime() == true);
> }
> ```

### Разбор

Способ через `builtin` работает на этапе компиляции — Zig знает целевую архитектуру заранее. Способ через память — классический runtime-трюк, который используют в системном программировании, когда нужно определить эндианность без стандартных средств.

Идея проста: `u16 = 0x0001`. Если первый байт в памяти равен `0x01` — машина little-endian (младший байт хранится по меньшему адресу). Если `0x00` — big-endian.

## Упражнение 3: Побитовые маски

**Сложность:** средний уровень

### Условие

Реализуй три функции для работы с отдельными байтами числа `u32`. Все операции должны работать только через `&`, `|`, `^`, `~` и hex-константы.

```
x = 0x87654321
         ^^ ^^ ^^ ^^
байт 3   байт 2  байт 1  байт 0 (младший)
  87       65       43       21
```

1. `loByte(x: u32) u32` — вернуть только младший байт, остальные занулить
   - `loByte(0x87654321)` → `0x00000021`

2. `invertUpperBytes(x: u32) u32` — инвертировать все байты **кроме** младшего
   - `invertUpperBytes(0x87654321)` → `0x789ABC21`

3. `setLoByte(x: u32) u32` — установить все биты младшего байта в `1`, остальные не трогать
   - `setLoByte(0x87654321)` → `0x876543FF`

**Требования:**
- Использовать только побитовые операции и маски-константы
- Тесты с `std.testing.expectEqual`

> [!tip]- Подсказка к loByte
> Маска, которая оставляет только младший байт: `0x000000FF`. Операция `AND` с маской обнуляет все биты вне маски.

> [!tip]- Подсказка к invertUpperBytes
> Инверсия `~` переворачивает все биты. Но тебе нужно инвертировать только старшие 3 байта, не трогая младший. Подумай: как скомбинировать инвертированную версию старших байтов с оригинальным младшим байтом? Операция `|` или `^`?

> [!tip]- Подсказка к setLoByte
> Маска `0x000000FF` устанавливает в `1` все биты младшего байта. Операция `OR` с маской устанавливает биты — а биты вне маски не меняются.

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
>
> // Оставить только младший байт, остальные обнулить
> fn loByte(x: u32) u32 {
>     return x & 0x000000FF;
> }
>
> // Инвертировать все байты кроме младшего
> fn invertUpperBytes(x: u32) u32 {
>     // Инвертируем всё: ~x
>     // Берём только старшие 3 байта из инвертированного: ~x & 0xFFFFFF00
>     // Берём только младший байт из оригинала: x & 0x000000FF
>     // Склеиваем: |
>     return (~x & 0xFFFFFF00) | (x & 0x000000FF);
> }
>
> // Установить все биты младшего байта в 1
> fn setLoByte(x: u32) u32 {
>     return x | 0x000000FF;
> }
>
> pub fn main() void {
>     const x: u32 = 0x87654321;
>     std.debug.print("x              = 0x{x:0>8}\n", .{x});
>     std.debug.print("loByte         = 0x{x:0>8}\n", .{loByte(x)});
>     std.debug.print("invertUpper    = 0x{x:0>8}\n", .{invertUpperBytes(x)});
>     std.debug.print("setLoByte      = 0x{x:0>8}\n", .{setLoByte(x)});
> }
>
> test "loByte" {
>     try std.testing.expectEqual(@as(u32, 0x00000021), loByte(0x87654321));
>     try std.testing.expectEqual(@as(u32, 0x000000FF), loByte(0xFFFFFFFF));
>     try std.testing.expectEqual(@as(u32, 0x00000000), loByte(0x12345600));
> }
>
> test "invertUpperBytes" {
>     // 0x87654321 -> старшие 3 байта инвертируются: 87->78, 65->9A, 43->BC
>     try std.testing.expectEqual(@as(u32, 0x789ABC21), invertUpperBytes(0x87654321));
>     // Младший байт не меняется
>     try std.testing.expectEqual(@as(u32, 0x000000AB), invertUpperBytes(0xFFFFFFAB));
> }
>
> test "setLoByte" {
>     try std.testing.expectEqual(@as(u32, 0x876543FF), setLoByte(0x87654321));
>     // Если младший байт уже FF — ничего не меняется
>     try std.testing.expectEqual(@as(u32, 0x000000FF), setLoByte(0x00000000));
> }
> ```

### Разбор

Три базовые маскирующие операции:

| Цель | Операция | Маска |
|---|---|---|
| Обнулить биты вне маски | `x & mask` | единицы там, где оставить |
| Установить биты маски в 1 | `x \| mask` | единицы там, где выставить |
| Инвертировать биты маски | `x ^ mask` | единицы там, где инвертировать |

`invertUpperBytes` — чуть сложнее: здесь нет одной операции, поэтому мы разбиваем задачу на два независимых региона (старшие 3 байта и младший байт), обрабатываем их отдельно и соединяем через `|`.

## Упражнение 4: bis и bic

**Сложность:** средний уровень

### Условие

В книге (задача 2.13) предлагается выразить стандартные операции через `bis` и `bic`:

- `bis(x, m)` — **bit set**: устанавливает в `x` все биты, которые равны `1` в маске `m`
- `bic(x, m)` — **bit clear**: сбрасывает в `x` все биты, которые равны `1` в маске `m`

**Шаг 1.** Реализуй `bis` и `bic` через стандартные побитовые операции.

**Шаг 2.** Реализуй через `bis` и `bic`:
- `boolOr(x, y) u32` — аналог `x | y`
- `boolXor(x, y) u32` — аналог `x ^ y`

Подсказка для XOR: `x ^ y = (x | y) & ~(x & y)` — но выразить это надо только через `bis` и `bic`, без прямого `|`, `^`.

**Тесты** для пар `(x, m)`:

| x | m | `bis` | `bic` |
|---|---|---|---|
| `0x12` | `0xF0` | `0xF2` | `0x02` |
| `0xFF` | `0x0F` | `0xFF` | `0xF0` |
| `0x69` | `0x55` | `0x7D` | `0x28` |

> [!tip]- Подсказка к bis
> `bis(x, m)` устанавливает биты маски — это операция `OR`.

> [!tip]- Подсказка к bic
> `bic(x, m)` сбрасывает биты маски. Нужно обнулить в `x` именно те позиции, где в `m` стоит `1`. Подумай: что делает `x & ~m`?

> [!tip]- Подсказка к boolXor
> `x ^ y` можно записать как: "биты, которые есть хотя бы в одном, но не в обоих". То есть `(x | y) & ~(x & y)`. Переведи это в термины `bis` и `bic`, помня что `bis` — это `|`, а `bic(x, m)` убирает из `x` биты `m`.

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
>
> // bit set: установить в x биты из m
> fn bis(x: u32, m: u32) u32 {
>     return x | m;
> }
>
> // bit clear: сбросить в x биты из m
> fn bic(x: u32, m: u32) u32 {
>     return x & ~m;
> }
>
> // x | y через bis
> fn boolOr(x: u32, y: u32) u32 {
>     return bis(x, y);
> }
>
> // x ^ y через bis и bic — только через них, без прямого &/|/^
> // bic(x, y) = биты, которые есть в x, но не в y
> // bic(y, x) = биты, которые есть в y, но не в x
> // Объединяем через bis — получаем ровно те биты, что есть ровно в одном = XOR
> fn boolXor(x: u32, y: u32) u32 {
>     return bis(bic(x, y), bic(y, x));
> }
>
> pub fn main() void {
>     const pairs = [_][2]u32{
>         .{ 0x12, 0xF0 },
>         .{ 0xFF, 0x0F },
>         .{ 0x69, 0x55 },
>     };
>     for (pairs) |p| {
>         const x = p[0];
>         const m = p[1];
>         std.debug.print(
>             "x=0x{x:0>2} m=0x{x:0>2}  bis=0x{x:0>2}  bic=0x{x:0>2}  or=0x{x:0>2}  xor=0x{x:0>2}\n",
>             .{ x, m, bis(x, m), bic(x, m), boolOr(x, m), boolXor(x, m) },
>         );
>     }
> }
>
> test "bis" {
>     try std.testing.expectEqual(@as(u32, 0xF2), bis(0x12, 0xF0));
>     try std.testing.expectEqual(@as(u32, 0xFF), bis(0xFF, 0x0F));
>     try std.testing.expectEqual(@as(u32, 0x7D), bis(0x69, 0x55));
> }
>
> test "bic" {
>     try std.testing.expectEqual(@as(u32, 0x02), bic(0x12, 0xF0));
>     try std.testing.expectEqual(@as(u32, 0xF0), bic(0xFF, 0x0F));
>     try std.testing.expectEqual(@as(u32, 0x28), bic(0x69, 0x55));
> }
>
> test "boolOr совпадает с |" {
>     try std.testing.expectEqual(0x12 | 0xF0, boolOr(0x12, 0xF0));
>     try std.testing.expectEqual(0x69 | 0x55, boolOr(0x69, 0x55));
> }
>
> test "boolXor совпадает с ^" {
>     try std.testing.expectEqual(0x12 ^ 0xF0, boolXor(0x12, 0xF0));
>     try std.testing.expectEqual(0xFF ^ 0x0F, boolXor(0xFF, 0x0F));
>     try std.testing.expectEqual(0x69 ^ 0x55, boolXor(0x69, 0x55));
> }
> ```

### Разбор

`bis` и `bic` — это примитивы, из которых можно собрать любую булеву операцию. В некоторых архитектурах (например, ARM) есть прямая инструкция `BIC`.

Вывод `boolXor` через `bis(bic(x,y), bic(y,x))` пошагово на примере `0x69 ^ 0x55`:
```
x        = 0110 1001
y        = 0101 0101
bic(x,y) = 0010 1000  ← биты x, которых нет в y
bic(y,x) = 0001 0100  ← биты y, которых нет в x
XOR      = 0011 1100  ← объединяем: bis(bic(x,y), bic(y,x))
```

## Упражнение 5: XOR-своп и баг из упражнения 2.11

**Сложность:** средний уровень

### Условие

**Часть A.** Реализуй `inplaceSwap(x: *u32, y: *u32)` — функцию обмена значений через XOR, без временной переменной.

Алгоритм:
```
x = x ^ y
y = x ^ y
x = x ^ y
```

**Часть B.** Реализуй `reverseArray(a: []u32)` через `inplaceSwap`. Функция должна разворачивать массив на месте.

Тесты:
- `{1, 2, 3, 4}` → `{4, 3, 2, 1}`
- `{1, 2, 3, 4, 5}` → `{5, 4, 3, 2, 1}`

**Часть C. Баг.** Что произойдёт, если передать `inplaceSwap` один и тот же адрес — `inplaceSwap(&a[i], &a[i])`?

Напиши тест, который демонстрирует этот баг: вызови `inplaceSwap(&val, &val)` и проверь, что значение обнулилось. Добавь комментарий, объясняющий почему.

> [!tip]- Подсказка к inplaceSwap
> В Zig `x.*` — это разыменование указателя. Запись `x.* = x.* ^ y.*` изменяет значение по адресу `x`.

> [!tip]- Подсказка к reverseArray
> Разворот массива — это обмен элементов с двух концов, движущихся навстречу. Первый и последний, второй и предпоследний, и так до середины. Индексы: `i` от `0` до `len/2 - 1`, второй индекс — `len - 1 - i`.

> [!tip]- Подсказка к багу
> Проследи алгоритм при `x` и `y` — один и тот же адрес. Шаг 1: `x = x ^ x`. Чему равно `a ^ a` для любого `a`?

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
>
> // Обмен значений через XOR без временной переменной
> fn inplaceSwap(x: *u32, y: *u32) void {
>     x.* = x.* ^ y.*;  // шаг 1
>     y.* = x.* ^ y.*;  // шаг 2
>     x.* = x.* ^ y.*;  // шаг 3
> }
>
> // Разворот массива на месте
> fn reverseArray(a: []u32) void {
>     var i: usize = 0;
>     while (i < a.len / 2) : (i += 1) {
>         inplaceSwap(&a[i], &a[a.len - 1 - i]);
>     }
> }
>
> pub fn main() void {
>     var arr = [_]u32{ 1, 2, 3, 4, 5 };
>     std.debug.print("До:    ", .{});
>     for (arr) |v| std.debug.print("{} ", .{v});
>
>     reverseArray(&arr);
>     std.debug.print("\nПосле: ", .{});
>     for (arr) |v| std.debug.print("{} ", .{v});
>     std.debug.print("\n", .{});
> }
>
> test "разворот массива чётной длины" {
>     var arr = [_]u32{ 1, 2, 3, 4 };
>     reverseArray(&arr);
>     try std.testing.expectEqualSlices(u32, &[_]u32{ 4, 3, 2, 1 }, &arr);
> }
>
> test "разворот массива нечётной длины" {
>     var arr = [_]u32{ 1, 2, 3, 4, 5 };
>     reverseArray(&arr);
>     try std.testing.expectEqualSlices(u32, &[_]u32{ 5, 4, 3, 2, 1 }, &arr);
> }
>
> test "баг XOR-свопа: одинаковый адрес обнуляет значение" {
>     var val: u32 = 42;
>
>     // Передаём один и тот же адрес в оба параметра.
>     // Шаг 1: val = val ^ val = 0  (любое число XOR себя = 0)
>     // Шаг 2: val = 0 ^ 0 = 0      (значение уже потеряно)
>     // Шаг 3: val = 0 ^ 0 = 0
>     //
>     // Это задача 2.11 из CS:APP: reverseArray на массиве нечётной
>     // длины в какой-то момент вызывает swap(a[mid], a[mid]) — и
>     // центральный элемент обнуляется вместо того, чтобы остаться.
>     inplaceSwap(&val, &val);
>
>     try std.testing.expectEqual(@as(u32, 0), val);
> }
> ```

### Разбор

**Почему XOR-своп работает в обычном случае:**

Для `x = A`, `y = B`:
```
x = A ^ B
y = (A ^ B) ^ B = A
x = (A ^ B) ^ A = B
```

**Почему он ломается при одинаковых адресах:**

Когда `x` и `y` указывают на одну и ту же ячейку памяти, шаг 1 вычисляет `val ^ val = 0` и записывает `0` в эту ячейку. Исходное значение уничтожено безвозвратно. Шаги 2 и 3 работают с нулём.

Это именно ошибка из упражнения 2.11 книги: `reverseArray` на массиве нечётной длины в какой-то итерации вызывает `swap(a[mid], a[mid])` — центральный элемент обнуляется.

**Вывод:** XOR-своп без временной переменной — это скорее любопытный трюк, чем практичный инструмент. Компилятор в любом случае оптимизирует обычный своп через temp-переменную до нескольких регистровых операций.

## Упражнение 6: Логический и арифметический сдвиг

**Сложность:** средний уровень

### Условие

Сдвиги влево и вправо работают по-разному для знаковых и беззнаковых типов.

**Часть A.** Реализуй функцию `shiftInfo` которая для заданного значения `u8` печатает:

```
=== 0xD4 (0b11010100) ===
<< 3  = 0b10100000 (0xA0)
>> 2  = 0b00110101 (0x35)  [логический,    u8]
>> 2  = 0b11110101 (0xF5)  [арифметический, i8]
```

Запусти для `0xD4` (`0b11010100`) и `0x63` (`0b01100011`).

**Часть B.** Напиши тест, который проверяет:
- `i8(-76) >> 4` = `-5` (арифметический сдвиг)
- `u8(0xB4) >> 4` = `0x0B` (логический сдвиг)

**Требования:**
- Для арифметического сдвига использовать тип `i8`
- Для форматирования двоичного числа использовать `{b:0>8}`

> [!tip]- Подсказка 1
> В Zig оператор `>>` для `u8` — логический (заполняет нулями), для `i8` — арифметический (заполняет знаковым битом). Нужно явно привести тип: `@as(i8, @bitCast(val)) >> 2`.

> [!tip]- Подсказка 2
> `@bitCast` позволяет переинтерпретировать биты без изменения значения — как `reinterpret_cast` в C++. `@as(i8, @bitCast(val_u8))` — переинтерпретирует байт `val_u8` как знаковое число (без изменения битов).

> [!tip]- Подсказка 3
> `-76` в двоичном для `i8` — это `0xB4 = 0b10110100`. Сдвиг вправо на 4 с заполнением знаковым битом даёт `0b11111011 = -5`.

### Решение

> [!warning]- Развернуть решение
> ```zig
> const std = @import("std");
>
> // Выводит информацию о сдвигах для заданного байта
> fn shiftInfo(label: []const u8, val_u8: u8) void {
>     const shl = val_u8 << 3;
>     const shr_logical = val_u8 >> 2;
>
>     // @bitCast переинтерпретирует биты u8 как i8 (без изменения битов)
>     // >> для i8 — арифметический: заполняет знаковым битом
>     const as_i8: i8 = @bitCast(val_u8);
>     const shr_arith: i8 = as_i8 >> 2;
>
>     std.debug.print("=== {s} (0b{b:0>8}) ===\n", .{ label, val_u8 });
>     std.debug.print("<< 3  = 0b{b:0>8} (0x{x:0>2})\n",                 .{ shl, shl });
>     std.debug.print(">> 2  = 0b{b:0>8} (0x{x:0>2})  [логический, u8]\n", .{ shr_logical, shr_logical });
>     // @bitCast обратно в u8 для единообразного вывода битов
>     const shr_arith_bits: u8 = @bitCast(shr_arith);
>     std.debug.print(">> 2  = 0b{b:0>8} (0x{x:0>2})  [арифметический, i8]\n\n", .{ shr_arith_bits, shr_arith_bits });
> }
>
> pub fn main() void {
>     shiftInfo("0xD4", 0xD4); // 0b11010100 — старший бит 1 (отрицательное как i8)
>     shiftInfo("0x63", 0x63); // 0b01100011 — старший бит 0 (положительное)
> }
>
> test "арифметический сдвиг i8(-76) >> 4 = -5" {
>     // -76 в дополнительном коде: 0b10110100 = 0xB4
>     // Сдвиг вправо на 4 с заполнением знаковым битом:
>     // 0b1111 1011 = -5
>     const val: i8 = -76;
>     const result = val >> 4;
>     try std.testing.expectEqual(@as(i8, -5), result);
> }
>
> test "логический сдвиг u8(0xB4) >> 4 = 0x0B" {
>     // 0b10110100 >> 4 = 0b00001011 = 0x0B
>     // Заполнение нулями слева — логический сдвиг
>     const val: u8 = 0xB4;
>     const result = val >> 4;
>     try std.testing.expectEqual(@as(u8, 0x0B), result);
> }
>
> test "сдвиг влево i8: тот же бит-паттерн что и у u8" {
>     // Сдвиг влево одинаков для знаковых и беззнаковых
>     const u: u8 = 0b00000001;
>     const i: i8 = 0b00000001;
>     const ru: u8 = u << 3;
>     const ri: i8 = i << 3;
>     const ri_bits: u8 = @bitCast(ri);
>     try std.testing.expectEqual(ru, ri_bits);
> }
> ```

### Разбор

Таблица сравнения для `0xD4 = 0b11010100`:

| Операция | Тип | Результат | Заполнение |
|---|---|---|---|
| `<< 3` | `u8` или `i8` | `0b10100000` | нулями справа (всегда) |
| `>> 2` | `u8` (беззнаковый) | `0b00110101` | нулями слева |
| `>> 2` | `i8` (знаковый) | `0b11110101` | знаковым битом (`1`) |

**Ключевое правило:**
- Сдвиг **влево** (`<<`) одинаков для знаковых и беззнаковых — справа добавляются нули.
- Сдвиг **вправо** (`>>`): для `u*` — логический (нули слева), для `i*` — арифметический (знаковый бит слева).

**Практическое значение:** `x >> k` для знакового типа — это деление на `2^k` с округлением вниз. `(-76) >> 4 = -76 / 16 = -4.75`, округлённое вниз = `-5`. Именно поэтому арифметический сдвиг полезен при работе со знаковыми числами.
