# 2.1.1 Шестнадцатеричная система счисления

## Главное

Hex используется потому что двоичное — громоздко, десятичное — неудобно конвертировать в биты.
Каждая hex-цифра = ровно **4 бита**. Это главное свойство.

## Hex → двоичное

Каждую цифру заменяешь на 4 бита.

```
0xC97B
C    9    7    B
1100 1001 0111 1011
```

## Двоичное → hex

Группируешь по 4 бита справа. Если крайняя левая группа неполная — дополняешь нулями слева.

```
1011 0110
B    6
= 0xB6

10 1101
→ 0010 1101
     2    D
= 0x2D
```

## Степени двойки → hex

2ⁿ в двоичном = единица и n нулей. Каждые 4 нуля = один hex 0.
Если n не кратно 4 — первая группа: 1→1, 10→2, 100→4, 1000→8.

```
2⁴  = 0x10
2⁸  = 0x100
2¹¹ = 0x800   (11 = 3 + 4·2, первая группа 1000 = 8)
2¹² = 0x1000
```
