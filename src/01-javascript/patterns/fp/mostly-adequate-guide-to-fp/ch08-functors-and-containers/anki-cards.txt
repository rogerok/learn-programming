Что такое функтор (Functor) простыми словами?	Функтор -- это контейнер (обёртка над значением), у которого есть метод map, соблюдающий два закона: идентичности и композиции. Array -- самый известный функтор в JavaScript.	fp::functors::basics
Какие два закона должен соблюдать функтор?	1) Закон идентичности: f.map(x => x) эквивалентен f. 2) Закон композиции: f.map(g).map(h) эквивалентен f.map(x => h(g(x))).	fp::functors::laws
Для чего нужен контейнер Maybe?	Maybe оборачивает значение, которое может быть null/undefined. Метод map пропускает трансформацию, если внутри null, позволяя строить цепочки без if-проверок.	fp::functors::maybe
Как безопасно извлечь значение из Maybe?	Через метод getOrElse(defaultValue), который возвращает значение из контейнера или указанное значение по умолчанию, если внутри null.	fp::functors::maybe
В чём разница между Maybe и Either?	Maybe говорит "есть значение или нет", но не объясняет причину отсутствия. Either хранит причину ошибки в Left и успешное значение в Right.	fp::functors::either
Что происходит с Left при вызове map?	Ничего -- map на Left игнорирует переданную функцию и возвращает тот же Left. Ошибка "проскальзывает" через всю цепочку.	fp::functors::either
Что делает метод fold у Either?	fold(fnLeft, fnRight) принимает две функции: первую вызывает для Left, вторую -- для Right. Позволяет обработать оба случая в одном месте.	fp::functors::either
Что хранит внутри контейнер IO?	IO хранит функцию (описание побочного эффекта), а не значение. Эффект не выполняется до явного вызова unsafePerformIO().	fp::functors::io
Зачем метод unsafePerformIO назван "unsafe"?	Чтобы подчеркнуть, что в этом месте происходит побочный эффект. В хорошо спроектированном приложении вызовы unsafePerformIO собраны на "краю" программы.	fp::functors::io
Какой общий паттерн у всех функторов (Container, Maybe, Either, IO)?	контейнер.map(fn) → новый контейнер того же типа. map применяет функцию к содержимому, не извлекая его и не меняя тип контейнера.	fp::functors::basics
Приведи пример, когда Maybe предпочтительнее Either.	При поиске элемента в массиве (find), доступе к необязательному полю объекта -- когда отсутствие значения нормально и причина очевидна.	fp::functors::maybe
Что общего между Array.map и Container.map?	Оба реализуют паттерн функтора: применяют функцию к содержимому контейнера и возвращают новый контейнер того же типа, не мутируя оригинал.	fp::functors::basics
