Чем декларативный стиль отличается от императивного?	Императивный описывает *как* (пошаговые инструкции: циклы, переменные). Декларативный описывает *что* (результат: map, filter, compose). Аналогия: SQL описывает *что* выбрать, а не *как* обойти таблицу.	fp::declarative::imperative
Какие четыре задачи решает FP-виджет из главы 6?	1) Сформировать URL. 2) Выполнить HTTP-запрос. 3) Извлечь и трансформировать данные. 4) Отобразить результат в DOM. Задачи 1 и 3 — чистые, 2 и 4 — побочные эффекты.	fp::example-app::architecture
Что такое паттерн Impure и зачем он нужен?	Объект-неймспейс, собирающий все побочные эффекты (fetch, DOM-мутации, console.log) в одном месте. Функции внутри каррированы для использования в compose-пайплайнах. Остальной код остаётся чистым.	fp::impure::side-effects
Почему функции в объекте Impure каррированы?	Чтобы можно было частично применить конфигурацию (callback, selector) и использовать результат в compose-пайплайне. Например, Impure.getJSON(render) возвращает функцию, ожидающую URL.	fp::impure::currying
Как называется архитектурный паттерн "чистые функции внутри, побочные эффекты снаружи"?	Functional core, imperative shell. Вся бизнес-логика живёт в чистых функциях (тестируемых без моков), а побочные эффекты вытеснены на границы приложения.	fp::architecture::functional-core
Запиши закон map composition law.	compose(map(f), map(g)) === map(compose(f, g)). Два последовательных map можно объединить в один map с композицией функций внутри.	fp::map::composition-law
Какое практическое преимущество даёт map composition law?	1) Оптимизация: один проход по массиву вместо двух. 2) Компактность: вместо двух отдельных трансформаций — одна объединённая. 3) Формально обоснованный рефакторинг.	fp::map::composition-law::optimization
Что делает функция trace в compose-пайплайне?	trace(tag, x) логирует значение x с меткой tag и возвращает x без изменений. Вставляется между функциями в compose для просмотра промежуточных значений без нарушения пайплайна.	fp::trace::debugging
Как в compose-пайплайне отладить промежуточное значение между функциями f и g?	Вставить Impure.trace('label') между ними: compose(f, Impure.trace('after g'), g). В консоли появится значение, которое g вернула и которое f получит.	fp::trace::compose::debugging
Почему compose(Impure.getJSON(render), url) работает как приложение?	При вызове app(albumId): 1) url(albumId) формирует URL-строку. 2) Impure.getJSON(render) — частично применённая функция — получает URL, делает fetch и передаёт JSON в render. Весь поток описан одной композицией.	fp::compose::example-app
Что произойдёт, если применить map composition law к compose(map(img), map(prop('thumbnailUrl')))?	Получится map(compose(img, prop('thumbnailUrl'))) — один проход по массиву, который сразу извлекает URL и создаёт тег img.	fp::map::composition-law::example
Почему Impure — это не полноценная изоляция побочных эффектов?	Impure лишь группирует и маркирует грязный код, но не делает его чистым. Побочные эффекты всё равно выполняются немедленно. Для полной изоляции в FP используют монаду IO, которая откладывает выполнение.	fp::impure::io-monad
