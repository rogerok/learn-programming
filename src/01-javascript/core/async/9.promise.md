# Промисы

Технически промис - это специальный объект, который отслеживает асинхронную операцию и хранит внутри себя её результат.
Он возвращается асинхронными ф-циями, построенных на промисах.

```ts
// Результат чтения файла передан в колбек-функцию, переданную в then
// колбек вызовется только тогда, когда выполнится чтение файла
fsp.readFile(src, "utf-8").then((content) => console.log(content));
```

Независимо от содержимого колбек-функции, вызов `then()` всегда возвращает новый промис.
А возврат колбек-функции становится доступным как параметр колбека следующего `then()`.

Если колбек вернет промис, то параметром следующего `then()` становится результат выполнения этого промиса, иначе промисы были б бесмысленны

```ts
const promise = /*тут какая-то операция*/ .then(() => fs.readFile(filePath)).then((text) => console.log(text))
```

Использование промисов внутри любой ф-ции автоматически делает эту ф-цию асинхронной.
Она больше не может вызываться как обычная синхронная ф-ция,
т.к. мы не сможем воспользоваться результатом ее работы.

```js
export const copy = (src, dest) => {
  return fps.readFile(src, "utf-8").then((content) => fsp.writeFile(dest, content));
};

// Использование

// делаем что-то синхронное

copy(src, dest)
  .then(() => {
    // делаем что-то еще
  })
  .then(/* продолжаем */)
  .then(/* продолжаем */);
```
