# Асинхронный код

В синхронном коде выполнение ф-ции происходит в том месте, где она было вызвана.
В асинхронном коде, вызов ф-ции не означает что она отработает прямо здесь и мы не знаем когда она отработает в принципе.

```ts
import fs from "fs";

// Обязательно передавать вторым параметром `utf-8`,
// только тогда данные возвратятся в строковом представлении
const content = fs.readFileSync("./myfile", "utf-8");
fs.writeFileSync("./myfile-copy", content);
```

При работе с `fs`, все что оканчивается на `Sync` вызывается синхронно и в данном случае, блокирует выполнение программы, т.е. программа ждет пока ОС прочитает файл и отдаст его содержимое.

Синхронный подход в работе с файловой системой занимает много времени, и неэффективно использует ресурсы.

Как выполнить данную ф-цию асинхронно?
Для этого можно использовать коллбеК, который будет вызван после того, как операция закончится.
Она имеет следующую сигнатуру

`callback(error, result)`

Первым параметром в нее передается ошибка, а втором результат операции, если все было хорошо.

```ts
import fs from "fs";

const cb = (_err, data) => console.log(data);

fs.readFile("./myfile", "utf-8", cb);
```

Запуск и вывод:

```
node index.js
content of file
```

Как только операция чтения файла выполнилась, интерпретатор Node.js внутри себя вызвал `cb`.

```ts
import fs from "fs";

const callback = (_error, data) => console.log(data);
console.log("before read");
// вызов функции не дожидается конца чтения файла,
// код сразу продолжит выполняться дальше
fs.readFile("./myfile", "utf-8", callback);
console.log("after read?");
```

Запуск и вывод

`node index.js
before read
after read?
content of file`

Хоть асинхронная ф-ция и запустилась сразу, но коллбек вызывается только тогда, когда в стеке не останется ни одной ф-ции.
В нашем случае это означает, что колбек запускается только после того, как отработает весь файл. И этот запуск породит свой собственный стек вызовов.

В асинхронном коде, каждый коллбек порождает свой стек вызовов, который в свою очередь, может выполнять асинхронные вызовы и так до бесконечности.

Node.js дожидается выполнения всех асинхронных вызовов, которые были сделаны в процессе работы программы

```ts
import fs from "fs";

fs.readFile("./myfile", "utf-8", (_error, data) => console.log("First!"));
fs.readFile("./myfile", "utf-8", (_error, data) => console.log("Second!"));
```

Запуск и вывод

```
node index.js

Second!
First!

node index.js

First!
Second!
```

В примере выше видно, что асинхронные операции могут выполниться в любом порядке, если они запускаются одновременно.

Единственный способ их упорядочить - делать последовательный запуск.
