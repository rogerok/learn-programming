# Таймеры

Функция, переданная в таймер, выполняется не в текущем стеке вызовов, а значит к таймерам применимы правила асинхронных
операций.
Ошибки в таймерах невозможно отследить с помощью `try/catch`, для этого нужно использовать колбеки.

Таймеры не гарантируют точный запуск через указанный промежуток времени.
Проверкой таймеров занимается райнтайм в тот момент, когда в текущем стеке вызвов не осталось кода.
Если запустить вычесление которое долго не выполняется, то все таймеры будут ждать пока оно закончится.

## Передать параметры в `setTimeout` можно тремя способами:

- Все аргументы, переданные в setTimeout после второго аргумента (времени), автоматически становятся аргументами
  функции, которую вызовет таймер.

```ts
const f = (a, b) => console.log(a + b);
setTimeout(f, 1000, 5, 8);
// =>  13
```

- callback
- bind

**Важно** понимать, что таймер не делает ф-цию асинхронной, а лишь откладывает ее выполнение. Если ф-ция переданная в
него синхронная, то она заблокирует основной поток выполнения.

Таймер можно остановить изнутри, использовав в колбеке его id.

```ts
let counter = 0;
const id = setInterval(() => {
  counter += 1;
  if (counter === 4) {
    clearInterval(id);
    return;
  }
  console.log(new Date());
}, 5000);
```
