# Ковариантность и контравариантность

## Вариантность — как направление совместимости типов зависит от позиции

### Ковариантность (выходная позиция — return)

Тип возвращаемого значения должен быть **таким же или уже**, чем ожидаемый.

```ts
type ComparatorCallback = (a: number, b: number) => -1 | 0 | 1;

// ❌ Ковариантная ошибка: number шире, чем -1 | 0 | 1
const bad: ComparatorCallback = (a, b) => Math.sign(a - b);
// Type 'number' is not assignable to type '-1 | 0 | 1'
```

`-1 | 0 | 1` ⊂ `number`, но ожидается `-1 | 0 | 1`, а передаётся `number` — расширение запрещено.

### Контравариантность (входная позиция — параметры)

Типы параметров должны быть **такими же или шире**, чем ожидаемые.

```ts
type Handler = (item: number) => void;

// ❌ Контравариантная ошибка: литерал 1 у́же, чем number
const bad: Handler = (item: 1) => console.log(item);
// Type 'number' is not assignable to type '1'
```

Функция с параметром `1` не может принять произвольный `number` — сужение параметров нарушает безопасность.

## Мнемоника

```
Параметры — контравариантны (принимают шире или равно)
Возврат    — ковариантен    (возвращает уже или равно)
```

## strictFunctionTypes

Флаг `strictFunctionTypes` (часть `strict`) включает контравариантную проверку параметров. Без него TypeScript проверяет бивариантно (и шире, и уже — разрешено), что небезопасно.

Исключение: **методы объектов** (`method(): void`) всегда проверяются бивариантно, даже с `strictFunctionTypes`. Поэтому для строгости предпочитайте свойство-функцию: `method: () => void`.

## Вариантность массивов

Массивы в TypeScript **ковариантны**, что технически небезопасно:

```ts
const dogs: Dog[] = [new Dog()];
const animals: Animal[] = dogs; // OK — ковариантно
animals.push(new Cat());        // Компилируется! Но dogs теперь содержит Cat
```

`readonly` массивы безопасно ковариантны, т.к. не допускают мутации.
