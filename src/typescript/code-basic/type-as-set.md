# Типы как множества


*Тип данных* — это множество всех значений и набор допустимых операций над ними.
С помощью типов мы накладываем ограничения на код. Например, если мы хотим работать с числами,
то нам нужно использовать операции сложения, вычитания и т.д. 
Если мы хотим работать со строками, то нам нужно использовать операции конкатенации, поиска подстроки и т.д


В TypeScript (и не только) тип можно представить как множество всех возможных значений, которые соответствуют этому типу.

- `string` — это бесконечное множество всех возможных строк: "", "hello", "привет", "123" и т.д.
- `number` — это множество всех чисел (включая `NaN`, `Infinity`).
- `boolean` — это множество из двух значений: { `true`, `false` }.

А что насчет более сложных типов?
* Литеральный тип 'hexlet' - это множество, содержащее всего *один* элемент
* `a | b` - множество из двух элементов "a", "b"

Раз типы — это множества, то к ним применимы операции из теории множеств. В TypeScript это напрямую отражено в синтаксисе.

1. Объединение (Union Type) - `|`- создаёт новое множество, которое включает все элементы из исходных множеств

2. Пересечение (Intersection Type) — `&` - создает новое множество, включающее только те элементы, которые одновременно принадлежат всем исходным множествам

3. Подмножество (Сужение / Subtyping) - один тип может быть *подмножеством* другого. Это значит, что все значения первого типа также являются значением второго типа.

* Тип `hello` является подмножеством типа `string`

Тип `never` используется в условных типах для *фильтрации* нежелательных типов.

```ts
// Утилита для отфильтровывания всех чисел из объединения
type NonNumbers<T> = T extends number ? never : T;
type Result = NonNumbers<string | number | boolean>; // Result = string | boolean
```
Поскольку `never` — пустое множество, он является подмножеством любого другого типа.
