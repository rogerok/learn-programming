# Что такое система типов

**Тип** — классификация данных, определяющая:
- Множество допустимых значений
- Множество допустимых операций над ними

**Type system** — набор правил, по которым язык назначает типы конструкциям программы (выражениям, переменным, функциям).

## Задачи системы типов

| Задача | Описание |
|---|---|
| **Safety** | Предотвращение ошибок. Type safety гарантирует: well-typed programs don't go wrong |
| **Абстракция** | Скрытие деталей реализации за интерфейсом типа |
| **Документация** | Сигнатуры типов — машинно-проверяемая документация |
| **Оптимизация** | Компилятор использует информацию о типах для генерации эффективного кода |

## Structural vs Nominal typing

| | Structural (TypeScript) | Nominal (Java, C#) |
|---|---|---|
| Совместимость | По форме (одинаковые поля) | По имени (явное наследование) |
| Гибкость | Высокая — duck typing | Строгая иерархия |
| Риск | Случайная совместимость разных доменов | Нужен boilerplate |

## Soundness vs Completeness

TypeScript **не является sound** — он допускает некоторые небезопасные операции (`any`, бивариантность методов, enum присваивание). Это осознанный trade-off: pragmatic type safety > теоретическая строгость.

Примеры unsoundness:
- `any` обходит все проверки
- Массивы ковариантны (мутабельные массивы должны быть инвариантны)
- `enum` значения присваиваются числам без проверки
