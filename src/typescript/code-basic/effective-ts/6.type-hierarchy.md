# Иерархия типов и проверки

## Полная иерархия

```
never                          — bottom type (∅)
  ↓
Literal Types (42, "hello")    — одноэлементные множества
  ↓
Primitive Types (number, string, boolean)
  ↓
Union Types (string | number)
  ↓
unknown                        — top type (универсальное множество)
any                            — вне системы типов
```

## Проверка через `extends`

`extends` в conditional types проверяет: является ли левый тип подтипом правого.

```ts
type A = 42 extends number ? 'YES' : 'NO';     // YES — 42 ⊂ number
type B = number extends 42 ? 'YES' : 'NO';     // NO  — number ⊄ 42
type C = never extends string ? 'YES' : 'NO';  // YES — ∅ ⊂ любое множество
```

## Excess Property Check

Дополнительная проверка **только для object literals** — ловит опечатки:

```ts
type Foo = { foo: string };

// ❌ Object literal — excess check активен
const a: Foo = { foo: 'hello', a: 3 }; // Error: unknown property 'a'

// ✅ Через переменную — excess check не работает
const obj = { foo: 'hello', baaar: 3 };
const b: Foo = obj; // OK — структурно совместим
```

Причина: TypeScript предполагает, что лишнее свойство в литерале — **опечатка**. При присваивании через переменную — объект мог прийти откуда угодно, и лишние поля ожидаемы.

## Почему excess check важен для опциональных полей

```ts
type Config = { host: string; port?: number };

const config: Config = { host: 'localhost', prot: 8080 };
// Error — поймали опечатку "prot" вместо "port"
```

Без excess check `prot` молча проигнорировалось бы, а `port` остался бы `undefined`.
