# Иерархия типов

## never

Иногда на практике нужно быть уверенным, что переменной не будет присвоено никакого значения. Это можно реализовать с помощью типа never:
Never - пустое множество, которое не содержит ни одного элемента.

```ts
let neverValue: never;
const two: 2 = 2;

neverValue = two; // Type 'number' is not assignable to type 'never'
```

Подтипы и супертипы

В TypeScript тип A является подтипом типа B, если значение типа A можно безопасно использовать там, где ожидается тип B.

Подтип (subtype) — более узкий, конкретный тип

Супертип (supertype) — более широкий, обобщённый тип

литеральный union:

```ts
type NumberUnion = -2 | -1 | 1 | 2
```

Это буквально подмножество number.

Следовательно:

`NumberUnion` — подтип

`number` — супертип

# Приведение типов

```ts
let num = 1; // Неявное восходящее приведение
let one: number = 1; // Явное восходящее приведение

let two = num as 2; // Явное нисходящее приведение

let three = 3 as const; // Приведение к литеральному типу — нисходящее
```

Когда мы присваиваем значение в переменную или передаем аргументы в ф-цию, 
TypeScript пытается сделать восходящие приведение - от подтипа к базовому.
Также можно явно задать восходящее приведение.

Приведение базового типа к подтипу делается явно с помощью ключевого слова `as`.
Приведение к нисходящему типу небезопасно - язык принимает поведение типов за истину.
