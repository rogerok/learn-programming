# Assignability (Присваиваемость)

Присваиваемость — проверка, можно ли значение одного типа использовать там, где ожидается другой.

## Правило через множества

Значение типа `A` присваивается типу `B`, если **множество значений `A` является подмножеством `B`**:

```ts
let x: number;
const y: 42 = 42;
x = y; // OK — {42} ⊂ number

let z: 42;
z = x; // Error — number ⊄ {42}
```

## Присваиваемость объектных типов

Объект с **дополнительными** полями присваивается типу с **меньшим** числом полей (больше полей = уже тип = подмножество):

```ts
type Named = { name: string };
const user = { name: 'Alice', age: 30 };
const named: Named = user; // OK — user имеет все поля Named + лишние
```

## Excess Property Check — исключение

При **прямом** присваивании литерального объекта TypeScript проверяет лишние поля:

```ts
const named: Named = { name: 'Alice', age: 30 }; // Error — excess property 'age'
```

Это **не** нарушение присваиваемости, а дополнительная проверка на опечатки, работающая **только для object literals**.

## Присваиваемость функций

Функция `f` присваивается типу `g`, если:
- Параметры `g` присваиваются параметрам `f` (**контравариантность**)
- Возврат `f` присваивается возврату `g` (**ковариантность**)
- `f` может иметь **меньше** параметров, чем `g` (лишние игнорируются)

```ts
type Handler = (a: number, b: string) => void;
const fn: Handler = (a) => {}; // OK — второй параметр можно не использовать
```
