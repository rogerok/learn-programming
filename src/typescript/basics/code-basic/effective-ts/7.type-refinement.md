# Type Refinement (Сужение типов)

## Вывод типов: const vs let

```ts
const a = 42;  // тип: 42 (literal) — значение не изменится
let b = 42;    // тип: number       — переменная может получить другое число
```

## Refinement — отсечение невозможных веток union

TypeScript анализирует control flow и **сужает** тип, исключая варианты, невозможные в текущей ветке:

```ts
function handle(value: string | number | null) {
  if (value === null) return;       // отсекли null
  if (typeof value === 'string') {
    value.toUpperCase();            // тип: string
  } else {
    value.toFixed(2);               // тип: number
  }
}
```

## Безопасные способы сужения

| Способ | Безопасность |
|---|---|
| `typeof` | Безопасен |
| `instanceof` | Безопасен |
| Дискриминатор (`type` поле) | Безопасен |
| Литеральное сравнение | Безопасно |
| Custom type guard (`is`) | Безопасен* (если логика верна) |

## Небезопасные способы сужения

### `in` оператор

```ts
type Internal = { name: string; department: string; subdepartment: string };
type External = { name: string; company: string };
type User = Internal | External;

function show(user: User) {
  if ('department' in user) {
    console.log(user.subdepartment.toUpperCase()); // user: Internal
  }
}

// Но External может тоже иметь поле department!
const obj = { name: 'John', company: 'Doe', department: 'S' };
const user: External = obj; // OK — структурная типизация
show(user); // Runtime Error: cannot read 'toUpperCase' of undefined
```

**Проблема**: структурная типизация не запрещает лишние поля, поэтому `External` может иметь `department`.

## Discriminated Unions — безопасный паттерн

Добавьте **литеральное поле-дискриминатор**, уникальное для каждого варианта:

```ts
type Event =
  | { type: 'login'; user: string }
  | { type: 'logout'; timestamp: number }
  | { type: 'payment'; amount: number };
```

## Exhaustive Check

```ts
function assertNever(x: never): never {
  throw new Error(`Unexpected: ${x}`);
}

function handle(e: Event) {
  switch (e.type) {
    case 'login': return e.user;
    case 'logout': return e.timestamp;
    default: assertNever(e);
    // Если добавить 'payment' в Event, но не обработать —
    // ошибка компиляции: Argument of type '{ type: "payment"; ... }'
    // is not assignable to parameter of type 'never'
  }
}
```
