# Generics и Overloads

## Generics — переменные для типов

```ts
function pick<Obj extends object, Key extends keyof Obj>(obj: Obj, key: Key) {
  return obj[key]; // тип возврата: Obj[Key]
}
```

### `extends` в generics — ограничение (constraint)

`extends` описывает **минимальные требования** к типу. Это определяет, какие операции допустимы:

```ts
function getLength<T extends { length: number }>(arg: T): number {
  return arg.length; // OK — T гарантированно имеет length
}
```

### Вывод generic-параметров

Для функций TypeScript выводит generics из **аргументов вызова** (а не из использования внутри тела):

```ts
pick({ name: 'Alice', age: 30 }, 'name');
// Obj = { name: string; age: number }, Key = 'name'
// Возврат: string
```

## `infer` — извлечение типа из структуры

`infer` работает только внутри conditional types. Он «захватывает» часть типа в переменную:

```ts
type Ref<Value> = { value: Value };
type MaybeRefOrCallable<Value> = Value | Ref<Value> | (() => Value);

// Извлекаем Value из обёртки
type ExtractValue<T> = T extends MaybeRefOrCallable<infer V> ? V : never;

type Result = ExtractValue<Ref<number>>; // number
```

### Встроенные утилиты на `infer`

```ts
// ReturnType — извлекает тип возврата функции
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

// Parameters — извлекает типы параметров как кортеж
type Parameters<T> = T extends (...args: infer P) => any ? P : never;

// Awaited — разворачивает Promise
type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T;
```

## Overloads vs Generics

Overloads — несколько сигнатур для одной функции:

```ts
function parse(input: string): string[];
function parse(input: string[]): string[][];
function parse(input: string | string[]): string[] | string[][] {
  // implementation
}
```

**Предпочитайте generics**, когда связь между входом и выходом выражается через один тип-параметр. Overloads — когда сигнатуры принципиально разные и generic не выражает зависимость.
