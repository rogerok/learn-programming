# Иерархия типов

## Полная иерархия (от узкого к широкому)

```
never                          — ∅ (пустое множество, bottom type)
  ↓
Literal Types (42, "hello")    — одноэлементные множества
  ↓
Primitive Types (number, string, boolean)
  ↓
Object Types ({ name: string })
  ↓
unknown                        — универсальное множество (top type)
any                            — выход из системы типов
```

## Подтип и супертип

- **Подтип (subtype)** — более узкий, конкретный: `42` ⊂ `number`
- **Супертип (supertype)** — более широкий, обобщённый: `number` ⊃ `42`

Правило: **подтип присваивается супертипу, но не наоборот.**

## Приведение типов (Type Casting)

```ts
let num = 1;            // Неявное восходящее: тип number (widening)
let one: number = 1;    // Явное восходящее
let two = num as 2;     // Явное нисходящее (downcasting) — небезопасно
let three = 3 as const; // Нисходящее к литеральному типу
```

- **Восходящее (upcasting)**: от подтипа к базовому — безопасно, происходит автоматически при присваивании и передаче аргументов.
- **Нисходящее (downcasting)** через `as`: небезопасно — компилятор доверяет программисту.

## never — bottom type

`never` присваивается любому типу (пустое множество — подмножество любого):

```ts
let x: never;
let s: string = x; // OK
let n: number = x; // OK
```

Но **ничто не присваивается `never`** (нет значений, принадлежащих пустому множеству).

## any — побег из системы типов

`any` **одновременно** ведёт себя как top type (принимает всё) и bottom type (присваивается всему). Это делает его **небезопасным** — он отключает проверку типов.

`unknown` — безопасная альтернатива: принимает всё, но требует сужения перед использованием.
