# Quick Reference: Claude Code + learn-programming

## Запуск

```bash
cd ~/path/to/learn-programming
claude
```

## 6 команд (через `/` в Claude Code)

| Команда               | Что делает                                                                    |
| --------------------- | ----------------------------------------------------------------------------- |
| `/plan Event Loop`    | Roadmap изучения темы с источниками                                           |
| `/chapter Event Loop` | Пошаговое создание главы (спрашивает перед каждым шагом — можно остановиться) |
| `/cards closures`     | Anki-карточки по теме                                                         |
| `/exercise generics`  | Задание с тестами                                                             |
| `/check`              | Проверить твоё решение (подсказки, не ответы)                                 |
| `/quiz prototypes`    | Ты объясняешь — Claude ищет пробелы                                           |

## 6 агентов (вызываются только когда ты просишь)

- **«запусти researcher по теме X»** — собирает источники и план
- **«используй author»** — пишет главу
- **«exercise-author, создай задание»** — упражнения с тестами
- **«запусти reviewer»** — проверка кода
- **«пусть methodist проверит»** — проверка педагогики
- **«exercise-checker, проверь решение»** — обратная связь по твоему коду

## Обычный чат (без агентов, без файлов)

- **«объясни X»** — ответ в чате
- **«напиши конспект по X»** — создаст .md файл в vault
- **«добавь в vault»** — сохранит в нужную папку

## Типичный цикл изучения

```
/plan новая тема           ← составляешь план
/chapter новая тема        ← проходишь шаги (останавливаешься когда хватит)
...решаешь упражнения...
/check                     ← получаешь обратную связь
/cards новая тема          ← карточки для Anki
...через пару дней...
/quiz новая тема           ← проверяешь, что помнишь
```

## Полезное

- `#` во время сессии — быстро добавить правило в CLAUDE.md
- Агенты не запускаются сами — токены тратятся только на то, что запросил
- `/chapter` спрашивает перед каждым шагом — можно пропустить или остановиться
- Подробности — в `SETUP.md`
